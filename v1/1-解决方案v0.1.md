# Solo AI 产品研发系统 —— 解决方案（Solution Design）

---

# 六、设计目标（Design Goals）

整个系统应满足以下目标：

---

## Goal 1：共享上下文（Shared Context）

任何 AI Agent 在进入项目时，能够立即获取完整项目上下文，包括：

- 产品目标
- 当前开发阶段
- Sprint 信息
- 当前任务（Task）
- 系统架构
- 技术规范
- 当前代码状态
- 测试状态

无需重复解释项目背景。

---

## Goal 2：减少人工切换（Reduce Manual Handoffs）

减少开发过程中因工具切换产生的人工操作。

避免频繁进行：

```text
ChatGPT
↓
复制 Prompt
↓
Codex
↓
复制结果
↓
ChatGPT
````

实现研发信息在各工具之间持续同步。

---


## Goal 3：统一项目状态（Single Source of Truth）

建立统一的项目状态管理机制。

确保任何时刻可以准确追踪：

```text
需求（Requirement）
↓
设计（Design）
↓
开发（Development）
↓
测试（Testing）
↓
Pull Request
↓
部署（Deployment）
```

所有阶段均具有明确对应关系。

---


## Goal 5：支持长期维护（Long-term Maintainability）

项目长期迭代后，任何 AI Agent 重新进入项目时，都能快速恢复完整项目上下文，而无需重新分析整个系统。

---

# 七、设计原则（Design Principles）

---

## Principle 1：Git Repository 是产品知识库

Git Repository 不仅用于代码管理，同时也是产品研发知识中心，包括：

* 产品文档
* 架构设计
* API 设计
* 开发任务
* 测试规范
* 技术决策
* 源代码

---

## Principle 2：GitHub 是统一状态中心

GitHub 作为统一状态管理中心，负责：

* 文档版本
* Issue
* Task
* Pull Request
* Code Review
* Release

所有 AI Agent 围绕 GitHub 获取项目状态。

---

## Principle 3：Markdown 是知识载体

Markdown 用于沉淀产品知识，而不是 ChatGPT 与 Codex 之间的临时传递文件。

所有设计文档应长期保存在仓库中并持续维护。

---

## Principle 4：职责明确

### ChatGPT

负责：

* 需求分析
* 产品设计
* 架构设计
* 技术方案
* Task 拆解
* Review
* 技术决策

---

### Codex

负责：

* 编码实现
* Bug 修复
* 测试
* 重构
* Pull Request

---

### GitHub

负责：

* 项目知识管理
* 状态同步
* 协作中心
* 版本管理

---

# 八、目标工作流（To-Be Workflow）

整体流程：

```text
                 Product Goal
                      │
                      ▼
             ChatGPT（PM / Architect）
                      │
      输出 PRD、Architecture、Task、Acceptance
                      │
                      ▼
          GitHub（Single Source of Truth）
                      │
      ┌───────────────┼────────────────┐
      ▼               ▼                ▼
  Codex                             ChatGPT
 (Coding)                           (Review)
      │                                │
      └───────────────┼────────────────┘
                      ▼
               Pull Request
                      ▼
             Review / Merge
                      ▼
              Deploy / Release
                      ▼
                 Next Sprint
```

* Pull Request
